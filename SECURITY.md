# Security policy

Do not report vulnerabilities in public issues. Report them privately using
the repository security-advisory feature or the maintainer contact in
[SUPPORT.md](SUPPORT.md), including reproduction steps, impact, and any safe
mitigation.

Maintainers will acknowledge reports within seven days, assess impact, and
coordinate a fix and disclosure timeline. Never include live credentials,
recovery kits, inventories, or customer infrastructure details in a report.

Supported releases are the latest published release and current prereleases
identified in GitHub Releases.
